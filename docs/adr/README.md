# Architecture Decision Records

An Architecture Decision Record (ADR) captures a decision that would be
expensive or confusing to rediscover later. Each ADR has a status of Proposed,
Accepted, Superseded, or Deprecated.

Use the next sequential number and this structure:

```md
# ADR-NNNN: Title

Status: Proposed
Date: YYYY-MM-DD

## Context

## Decision

## Consequences

## Alternatives considered
```

Product requirements belong in the other `docs/` files. ADRs record the
technical or durable product decision used to implement them.
